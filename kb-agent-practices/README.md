# kb-agent-practices

How to work with coding agents to produce good results. Process patterns, not code patterns: how to structure specs, code, and review loops so agent output stays traceable, auditable, and worth keeping. ergo is the reference manifestation; its practices seeded this kb.

## Contents

- `decision-hygiene.md` - greppable decision comments, tenets cross-linked to code
- `distillation.md` - extracting differentiated knowledge from a codebase with an agent
- `lock-in.md` - inverted orchestration: agent as PM, human as execution multiplexer; executable form in juan-skills

## When to pull into a workspace

Any workspace where agents write or review code. That is most of them, so the bar for content here is high: only practices proven on real work.
